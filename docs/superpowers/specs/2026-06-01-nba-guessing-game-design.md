# NBA 球员猜词游戏 — 设计文档

> **日期：** 2026-06-01  
> **平台：** 微信小程序  
> **状态：** 设计已确认，待实现

---

## 1. 项目概述

一款基于多维相似度的 NBA 球员猜词游戏。系统随机抽取一名目标球员，玩家通过输入球员姓名进行猜测。每次猜错后，系统给出一个**相似度百分比**（基于 15 个维度加权计算）和**一条关键提示**，历史猜测按相似度从高到低排列，引导玩家逐步推理出正确答案。

### 核心体验

- 不是盲猜，而是通过不断收窄范围的有推理感的过程
- 每一条提示都是可解释的（"曾和韦德在热火当过队友"）
- 玩家对 NBA 的了解程度直接影响猜测效率

## 2. 功能清单

### 2.1 游戏模式

| 模式 | 说明 |
|------|------|
| **每日挑战** | 每天固定一个球员，当天所有人都猜同一个 |
| **无限模式** | 每局随机抽一个球员，猜完可立即开下一局 |

### 2.2 难度档位

| 难度 | 球员范围 | 人数约 |
|------|---------|--------|
| ⭐ 球星 | 全明星 ≥ 3 次 或 入选过最佳阵容 | ~200-300 |
| ⭐⭐ 知名 | 首发级别、场均得分 > 10 或出场数够多 | ~500-800 |
| ⭐⭐⭐ 全部 | 数据库中所有球员 | ~1000+ |

### 2.3 核心交互

1. 首页选择模式 + 难度
2. 进入游戏页，系统随机抽取答案球员（不在 UI 中显示）
3. 输入框输入球员姓名（模糊匹配、自动补全），点击提交
4. 系统计算 15 维相似度，若猜中则结束，若未猜中则：
   - 显示相似度百分比
   - 给出得分最高维度的提示文字
   - 历史猜测列表按相似度由高到低重新排序
5. 猜中后展示：猜测次数、答案球员全部信息

## 5. 相似度算法

### 5.1 15 个维度及权重

| # | 维度 | 权重 | 计算方式 |
|---|------|------|---------|
| 1 | 位置 | 0.10 | 同位置=1.0，相邻位置=0.5，否则=0 |
| 2 | 身高体重 | 0.08 | 高斯衰减，差值越小分越高 |
| 3 | 队友关系 | 0.15 | 同一球队+时间重叠 → 1.0 |
| 4 | 同队轨迹 | 0.10 | 不同时期效力过同一球队 → 0.8 |
| 5 | 季后赛交手 | 0.10 | 季后赛对阵过 → 1.0 |
| 6 | 荣誉匹配 | 0.08 | MVP/冠军/全明星次数接近程度 |
| 7 | 选秀 | 0.05 | 同年+顺位接近 → 1.0，仅同年=0.5 |
| 8 | 球衣号码 | 0.04 | 共用过同一号码=1.0 |
| 9 | 生涯时段 | 0.06 | 打球年份重叠度（Jaccard 相似度） |
| 10 | 国籍/大学 | 0.04 | 同国=0.6，同大学=0.8，都有=1.0 |
| 11 | 全明星级别 | 0.06 | 全明星次数同档（0,1-3,4-7,8+） |
| 12 | 球风 | 0.06 | 同一球风分类=1.0 |
| 13 | 名人堂 | 0.03 | 同为名人堂/同不为名人堂=1.0 |
| 14 | 生涯数据量级 | 0.03 | 得分/篮板/助攻各分 5 档，取平均 |
| 15 | 知名关联 | 0.02 | 交易互换/宿敌/师徒=1.0 |

### 5.2 公式

```
similarity = Σ (dimension_score_i × weight_i) × 100%
```

每个 `dimension_score` 范围 [0, 1]，最终结果映射为百分比。

### 5.3 提示生成规则

取**得分最高且 ≥ 0.5** 的维度，映射为自然语言提示：

| 维度 | 高分时提示模板 |
|------|--------------|
| 队友 | "曾与 {name} 在 {team} 当过队友" |
| 同队轨迹 | "也曾效力于 {team}" |
| 位置 | "位置相同：{position}" 或 "位置接近：{pos1} 和 {pos2}" |
| 季后赛交手 | "曾在 {year} 年季后赛交手" |
| 荣誉 | "荣誉级别接近（MVP/冠军数相当）" |
| 选秀 | "同届选秀（{year} 年）" |
| 球衣号码 | "曾穿同一个号码 #{num}" |
| 生涯时段 | "职业生涯同时期" |
| 国籍/大学 | "同样来自 {country}" 或 "同为 {college} 校友" |
| 全明星 | "全明星次数相近" |
| 球风 | "球风相似：{style}" |
| 名人堂 | "同为名人堂成员" |
| 体型 | "身高体重接近" |
| 生涯数据 | "生涯场均数据相近" |
| 知名关联 | 直接展示 relation 字段内容 |

### 6.3 UI 交互细节

- 输入框支持**模糊匹配 + 下拉补全**，确保提交的名字是有效球员
- 新猜测插入时，列表**实时按相似度降序排列**
- 相似度用颜色区分热度：>70% 红色，40-70% 橙色，<40% 灰色

## 8. 约束与边界

- **不在 v1 中做：** 排行榜、分享功能、多语言、音效、动画
- **队友关系 / 季后赛交手数据** — 优先尝试爬取，若数据源不可用则降低权重或暂时用占位值
- **每日挑战的"天"** — 基于 UTC+8 日期，种子由日期字符串哈希确定
- **小程序代码包上限 2MB** — 球员数据预计 500-800KB，绰绰有余
